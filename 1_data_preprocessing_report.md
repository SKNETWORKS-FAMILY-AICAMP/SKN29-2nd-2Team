# 📄 산출물 1: 인공지능 데이터 전처리 결과서

## 1. 프로젝트 개요

### 1-1. 비즈니스 배경
- 신용카드 이상거래는 고객의 직접적인 금전 피해와 금융사의 운영 리스크로 이어질 수 있다.
- 기존 Rule 기반 탐지는 명확한 조건을 빠르게 판단할 수 있지만, 새로운 이상거래 패턴이나 복합적인 위험 요인을 반영하는 데 한계가 있다.
- 본 프로젝트는 신용카드 거래 데이터를 기반으로 이상거래를 탐지하고, 고위험 거래를 사전에 선별하는 것을 목표로 한다.

### 1-2. 비즈니스 목표
- 이상거래 가능성이 높은 거래를 조기에 탐지한다.
- 정상 거래를 과도하게 차단하지 않으면서 실제 Fraud 거래의 미탐을 줄인다.
- 거래별 위험도를 점수화하여 `Pass`, `Review`, `Block`과 같은 운영 판단에 활용한다.

### 1-3. 머신러닝 활용 계획
| 항목 | 내용 |
|------|------|
| 문제 유형 | 이진 분류 |
| 예측 대상 | 신용카드 거래의 이상거래 여부 |
| Target | `is_fraud` |
| 정상 거래 | `0` |
| 이상거래 | `1` |
| 활용 방안 | 이상거래 탐지, 고위험 거래 선별, 대시보드 시각화, Threshold 시뮬레이션 |
| 기대 효과 | Fraud 거래 조기 탐지, 미탐 감소, 오탐 관리, 운영 의사결정 지원 |

### 1-4. 성공 기준 (Success Criteria)
- 모델 성능 기준: Recall, F1 Score, F2 Score, PR-AUC 중심 평가
- 비즈니스 기준: 실제 이상거래를 최대한 놓치지 않으면서 정상 거래 오탐을 관리할 수 있는 탐지 구조 설계
- 운영 기준: Risk Score, ML Score, Anomaly Score를 결합한 Final Score 기반 고위험 거래 선별

## 3. 탐색적 데이터 분석 (EDA)

### 3-1. 기초 통계량
- 거래 금액(`amt`)은 정상 거래와 이상거래 간 차이를 확인할 수 있는 주요 변수이다.
- 거래 시간, 업종, 위치 정보는 이상거래 패턴 분석에 활용 가능하다.
- 전체 데이터는 정상 거래가 대부분이며, Fraud 거래는 매우 적은 비율로 존재한다.

### 3-2. 결측값 분석
| 특성명 | 결측 수 | 결측률(%) | 처리 방안 |
|--------|--------:|----------:|----------|
| 주요 학습 컬럼 | 0 | 0% | 별도 대체 처리 없음 |

- 주요 학습 컬럼 기준 결측치는 확인되지 않았다.
- 결측치가 없는 상태에서 전처리 및 Feature Engineering을 진행하였다.

### 3-3. 이상값 분석
| 특성명 | 이상값 판단 기준 | 처리 방법 | 근거 |
|--------|----------------|----------|------|
| `amt` | 거래 금액의 극단값 | 제거하지 않고 활용 | 고액 거래는 Fraud의 주요 신호일 수 있음 |
| `distance_km` | 고객-가맹점 간 장거리 거래 | 제거하지 않고 활용 | 비정상 지역 거래 탐지에 필요 |
| `trans_hour` | 심야 시간대 거래 | 제거하지 않고 활용 | 시간대 기반 Fraud 패턴 분석에 필요 |
| `amt_zscore` | 고객 평균 대비 큰 금액 편차 | 파생 변수로 활용 | 평소 소비 패턴 이탈 탐지 |

- 이상거래 탐지 문제에서는 이상값 자체가 중요한 탐지 신호가 될 수 있다.
- 따라서 이상값을 단순 제거하지 않고, 위험도를 설명하는 파생 변수로 활용하였다.

### 3-4. 특성별 분포 분석
- 금액 기반 분석: 고액 거래에서 이상거래 가능성이 높아질 수 있음을 확인하였다.
- 시간 기반 분석: 심야 시간대 거래를 별도로 구분할 필요가 있음을 확인하였다.
- 거리 기반 분석: 고객 위치와 가맹점 위치 간 거리가 먼 거래를 위험 신호로 활용할 수 있음을 확인하였다.
- 업종 기반 분석: 특정 업종에서 Fraud 패턴이 다르게 나타날 가능성이 있어 `category` 변수를 모델 입력에 포함하였다.

### 3-5. 상관관계 분석
- 수치형 변수 간 상관관계를 확인하여 중복성이 큰 변수 여부를 검토하였다.
- `amt`, `distance_km`, `amt_zscore`, `hour_dev` 등은 Fraud 탐지에 활용 가능한 변수로 판단하였다.
- 다중공선성이 큰 변수는 모델 성능 및 해석에 영향을 줄 수 있으므로 최종 Feature 구성 시 검토하였다.

### 3-6. EDA 주요 발견사항 (Insight)
- 발견 1: Fraud 거래는 정상 거래보다 높은 금액 또는 평소와 다른 금액 패턴에서 발생할 가능성이 있다.
- 발견 2: 심야 시간대 거래는 이상거래 탐지에서 유의미한 위험 요인으로 활용 가능하다.
- 발견 3: 고객 위치와 가맹점 위치 간 거리가 먼 거래는 비정상 거래 패턴으로 볼 수 있다.
- 발견 4: 단일 조건만으로 Fraud를 판단하기 어렵기 때문에 금액, 시간, 거리 정보를 결합한 복합 Feature가 필요하다.

## 4. 데이터 전처리

### 4-1. 결측값 처리
| 특성명 | 처리 방법 | 근거 |
|--------|----------|------|
| 주요 학습 컬럼 | 별도 처리 없음 | 결측치가 확인되지 않음 |

### 4-2. 이상값 처리
| 특성명 | 탐지 방법 | 처리 방법 | 처리 전 범위 | 처리 후 범위 |
|--------|----------|----------|------------|------------|
| `amt` | 기초 통계 및 분포 확인 | 제거하지 않고 `amt_zscore` 생성에 활용 | 원본 유지 | 원본 유지 |
| `distance_km` | 고객-가맹점 거리 계산 | 제거하지 않고 거리 기반 Feature로 활용 | 생성 전 | 생성 후 활용 |
| `trans_hour` | 거래 시간 분포 확인 | 심야 여부 `is_night` 생성 | 0~23 | 0~23 유지 |
| `hour_dev` | 평균 거래 시간 대비 편차 | 시간 패턴 이탈 변수로 활용 | 생성 전 | 생성 후 활용 |

### 4-3. 인코딩 (범주형 → 수치형)
| 특성명 | 인코딩 방법 | 변환 결과 | 선택 근거 |
|--------|------------|----------|----------|
| `gender` | Label Encoding | M/F → 수치형 값 | 이진 범주형 변수 |
| `category` | One-Hot Encoding 또는 모델 파이프라인 내 변환 | 업종별 수치형 Feature | 명목형 변수 |
| `state` | One-Hot Encoding 또는 모델 파이프라인 내 변환 | 지역별 수치형 Feature | 명목형 변수 |
| `merchant` | 모델 입력 형식에 맞게 변환 | 가맹점 정보 반영 | 거래처 패턴 반영 |

### 4-4. 특성 공학 (Feature Engineering)
| 생성 특성 | 생성 근거 | 활용 목적 |
|----------|----------|----------|
| `trans_hour` | 거래 일시에서 시간 추출 | 시간대별 Fraud 패턴 분석 |
| `trans_dayofweek` | 거래 일시에서 요일 추출 | 요일별 거래 패턴 분석 |
| `trans_month` | 거래 일시에서 월 추출 | 월별 거래 패턴 분석 |
| `trans_day` | 거래 일시에서 일 추출 | 일자별 거래 패턴 분석 |
| `age` | 생년월일 기반 계산 | 고객 특성 반영 |
| `distance_km` | 고객-가맹점 좌표 기반 계산 | 장거리 거래 위험 탐지 |
| `amt_zscore` | 고객 평균 대비 거래 금액 표준화 | 평소 대비 고액 거래 탐지 |
| `hour_dev` | 평균 거래 시간 대비 편차 | 비정상 시간 거래 탐지 |
| `high_amt_far` | 고액 + 장거리 조건 | 복합 위험 거래 탐지 |
| `is_night` | 심야 시간 여부 | 심야 거래 위험 반영 |

### 4-5. 불균형 데이터 처리
| 방법 | 적용 여부 | 처리 전 비율 | 처리 후 비율 |
|------|----------|------------|------------|
| SMOTE | 미적용 | Train Fraud 약 0.58% | 동일 |
| class_weight / scale_pos_weight | 적용 검토 및 일부 모델 반영 | Train Fraud 약 0.58% | 동일 |
| 언더샘플링 | 미적용 | Train Fraud 약 0.58% | 동일 |
| Threshold 조정 | 적용 | 기본 threshold 기준 | F2 기준 최적 threshold 탐색 |

- 데이터 자체를 인위적으로 크게 변경하기보다, 모델 학습 및 평가 단계에서 불균형 문제를 고려하였다.
- 평가 지표는 Accuracy보다 Recall, F1, F2, PR-AUC를 중심으로 확인하였다.

### 4-6. 스케일링
| 방법 | 적용 특성 | 선택 근거 |
|------|----------|----------|
| StandardScaler | AutoEncoder 입력 수치형 변수 | Reconstruction Error 기반 이상 탐지에 필요 |
| 별도 스케일링 미적용 | 트리 기반 모델 입력 일부 | 트리 기반 모델은 스케일 영향이 상대적으로 작음 |

### 4-7. 불필요 특성 제거
| 제거 특성 | 제거 근거 |
|----------|----------|
| `first`, `last`, `street`, `city`, `job` 등 개인정보성 또는 직접 예측 기여가 낮은 컬럼 | 개인정보성 정보 또는 모델 일반화에 불필요 |
| `trans_date_trans_time` | 시간 파생 변수 생성 후 원본 일시 컬럼은 직접 학습에서 제외 가능 |
| `dob` | `age` 생성 후 직접 학습에서는 제외 가능 |
| `trans_num` | 거래 고유 식별자이며 예측 변수로는 부적합 |

### 6-2. 전처리 파이프라인 요약

```text
    ├─ 고객 나이(age) 생성
    ├─ 고객-가맹점 거리(distance_km) 계산
    ├─ 거래 금액 편차(amt_zscore) 생성
    ├─ 시간 편차(hour_dev) 생성
    ├─ 심야 거래 여부(is_night) 생성
    ├─ 고액+장거리 거래 여부(high_amt_far) 생성
    ├─ 범주형 변수 인코딩
    ├─ AutoEncoder 입력용 스케일링
    └─ Risk / ML / Anomaly / Final Score 연계

[전처리 완료 데이터] → 모델 학습 및 대시보드 활용
````
